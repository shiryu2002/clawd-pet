# Clawd ドット絵エディタ

Clawd ペット（`clawd-pet.mjs` の `STAGES`）のドット絵を、ブラウザ上で編集・保存し、
良いものを選んでコードに反映するためのローカル Web アプリ。

本体のアートは 4 象限ブロック文字（`▘▝▖▗▌▐▀▄█…`）でできている。1 文字が 2×2 の
サブピクセルに対応するので、このエディタでは「ドット（サブピクセル）」のグリッドとして
編集し、保存・適用の直前にブロック文字へ畳み込む。

## 起動

```bash
npm run editor          # = node clawd-web-editor/server.mjs
```

起動したら http://127.0.0.1:4173 をブラウザで開く。ポートは `PORT` 環境変数で変えられる。

> ブラウザは ES module を読み込むため、必ずサーバー経由で開くこと（`file://` 直開きは不可）。

## 使い方

1. **成長ステージ**タブで編集したいステージを選ぶ（clawd 〜 でんせつのclawd）。
2. **パーツ**タブでフレーム／まばたきを切り替える。
3. キャンバスをドラッグしてドットを描く。右ドラッグまたは消しゴムで消去。
4. **列／行**でグリッドサイズ、**拡大**で表示倍率を調整。
5. **このステージの設定**で名前・`bellyRow`・`crownRows`・`noBreath`／`petOpen` を編集。
6. 右上の**プレビュー**でアニメーションを確認。

ツール: ペン／消しゴム／全消去／左右反転／他パーツから複製。

## 保存と適用は別

要件どおり、**溜める**操作と**コードへ反映する**操作を分けてある。

- **保存（溜める）** — 現在の全ステージを 1 つのデザインとして `designs/` に JSON で保存する。
  いくつでも溜められる。`JSON 書き出し／読み込み`でファイル単体のやり取りも可能。
- **コードに適用** — 選んだデザイン（または編集中の内容）を `clawd-pet.mjs` の `STAGES`
  定義へ書き込む。保存済みデザインの各行「適用」からも、編集中の内容の「適用」からも実行できる。

つまり「たくさん保存 → 見比べて → 良いものだけ適用」という流れになる。

### 適用の安全機構

`POST /api/apply` は本体を壊さないよう次の順で動く。

1. `STAGES` 定義だけを差し替えた新しいソースを生成する。
2. 一時ファイルに書き出し `node --check` で構文チェックする。
3. 通った場合のみ、`clawd-pet.mjs.bak-<日時>` にバックアップを取ってから上書きする。

チェックに失敗したら本体には一切触れない。バックアップは `.gitignore` 済み。

> 注意: 適用すると `STAGES` 定義内の手書きコメントは失われる（アートとメタ情報は保持される）。

## 構成

| ファイル | 役割 |
| --- | --- |
| `index.html` | エディタ本体（依存なしの vanilla JS） |
| `codec.mjs` | ブロック文字 ⇄ ドットグリッド変換、`STAGES` ソースの生成・置換（純粋関数） |
| `server.mjs` | ローカルサーバー（依存ゼロ）。取得 / 保存 / 一覧 / 削除 / 適用の API |
| `codec.test.mjs` | ラウンドトリップとソース置換の健全性テスト |
| `designs/` | 保存したデザイン（git 管理外） |

`codec.mjs` はブラウザと Node の双方から import される。テストは `npm test` に含まれる。

## API（ローカル専用）

| メソッド | パス | 内容 |
| --- | --- | --- |
| `GET` | `/api/stages` | 現在の `STAGES` を返す |
| `GET` | `/api/designs` | 保存済みデザイン一覧 |
| `POST` | `/api/designs` | デザインを保存（溜める） |
| `DELETE` | `/api/designs/:id` | デザインを削除 |
| `POST` | `/api/apply` | デザインを `clawd-pet.mjs` に適用 |

環境変数 `CLAWD_PET_FILE` / `CLAWD_DESIGNS_DIR` で対象ファイルと保存先を差し替えられる（テスト用）。
