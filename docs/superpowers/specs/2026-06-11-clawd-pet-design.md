# clawd-pet 設計

Claude Code の日次消費トークン数で成長する clawd を、ターミナルウィンドウにリアルタイム表示する常駐スクリプト。

## 目的と要件

- Claude Code のマスコット clawd が、その日の消費トークン量に応じて育つ
- 専用ターミナルウィンドウで常時実行する。アニメーションは常時動き、トークン集計は 3 分間隔で更新する
- 集計対象はその日（JST）の消費のみ。日付が変わると初期状態に戻る
- トークンの定義: input + output + cache creation + cache read の合計（ccusage の daily と同じ定義）
- 成長表現: 段階進化。デフォルトの clawd が初期状態で、段階が上がると大きさも形も変わる
- 表示要素: clawd 本体、成長ゲージ、今日のトークン数値（M表記）、今日のコスト（USD）、アイドルアニメ、一言セリフ、消費ペース

## 構成

```
clawd-pet/
├── clawd-pet.sh    # 起動ラッパー（exec node clawd-pet.mjs）
├── clawd-pet.mjs   # 本体（依存ゼロ・単一ファイル）
└── test/
    └── core.test.mjs  # node:test によるユニットテスト
```

本体内部は3つの関心に分離する（単一ファイル内の関数群として）。

- **collector** — `~/.claude/projects/**/*.jsonl` の集計。状態は「ファイルごとの読み取りオフセット」と「日付ごとの合計」のみ
- **state** — ステージ判定、ペース計算、セリフ選択。純粋なロジック（入力 → 出力のみ、I/Oなし）
- **renderer** — ANSI 描画。state の出力を受け取って画面文字列を組み立てる

## データソース

`~/.claude/projects/<プロジェクト>/<セッションID>.jsonl` の各行に、assistant メッセージごとの usage が記録されている。

```json
{"timestamp":"...","requestId":"...","message":{"id":"...","usage":{
  "input_tokens":2,"cache_creation_input_tokens":1660,
  "cache_read_input_tokens":31440,"output_tokens":313}}}
```

実測（2026-06-11、47プロジェクト）: 全走査 0.8 秒。日次合計はアクティブな日で 1,000万〜1億、最大 3.5 億トークン。

## データフロー

- **起動時**: 今日（JST 0時）以降に mtime があるファイルを全走査し、今日の合計をシード
- **以後**: ペースが算出できるまで（起動直後・日付またぎ直後）は 60 秒ごと、算出できたら 3 分ごとに、projects 配下を列挙し、各ファイルの現サイズと前回オフセットを比較。追記があったファイルだけ追記分を読む。新規ファイルは先頭から読む
- **行の処理**: `"usage"` を含む行のみ JSON.parse。壊れた行はスキップ。`message.id + requestId` をキーに重複排除（保持は当日分のみ）
- **日付またぎ**: タイムスタンプを JST の日付キーに変換して集計。表示は描画時に「今日」のキーを引き直すため、0 時を過ぎた瞬間にゼロ表示へ戻る。前日分の dedup セットと合計は次回スキャン時に破棄

## 成長ステージ

実績に基づく対数スケールで 6 段階。閾値は当日合計トークン（キャッシュ込み）。

| Stage | 閾値 | 姿 |
|---|---|---|
| 1 | 0 | デフォルト clawd（本家風の小さい姿） |
| 2 | 200万 | 少し大きく |
| 3 | 1,000万 | 形が変わる |
| 4 | 3,000万 | とんがり耳＋しっぽ（しっぽふりアニメ） |
| 5 | 8,000万 | 耳＋ふさふさしっぽ |
| 6 | 2億 | 浮き王冠（金・5マス）＋耳＋しっぽ |

普段の日は Stage 3〜4、重い日に 5、最大級の日だけ 6 に届く配分。

各ステージのアートは、通常 2 フレーム（アイドルアニメ用）＋まばたき 1 フレーム（全ステージ）。体色は本家のオレンジ（truecolor `#D97757`、非対応端末は 256 色の 208 にフォールバック）。Stage 6 の王冠（アート先頭 2 行）は金色で描く。

## 睡眠

トークン消費が 15 分間動かないと眠る。寝姿はまばたきフレーム＋暗めの体色で、2 秒周期でお腹の行が 1 行抜けて体が縮む（呼吸。各ステージの bellyRow から上を4象限合成で半行シフト、下揃えなので足は接地したまま）。吹き出しの代わりに頭の右上へ Zzz が 1 秒周期で積み上がる。消費が再開した次のスキャンで起きる。`--sleep` フラグで試し見できる。

## 画面レイアウト

```
       ╭──────────────────────────────╮
       │ きょうも いっしょに がんばろ │
       ╰──────────────────────────────╯
                ▐▛███▜▌
               ▝▜█████▛▘
                 ▘▘ ▝▝

  Stage 1 ── clawd
  Today : 18,339,198 tokens
  0 [█████████████░░░░░░░░░] 2M → Stage 2
  Cost  : $31.44
  Pace  : 4.2M tokens/h
```

吹き出しとペットは共通の中心軸に揃え、吹き出しはペットの頭のすぐ上に付ける。描画時にコンテンツ全体（上部の成長余白は除く）を端末の縦横中央に配置する。

- 代替スクリーンバッファ（`\x1b[?1049h`）を使い、Ctrl+C / SIGTERM で元の画面に復帰
- 描画は 500ms tick（アニメーション）、データ更新は 3 分 tick
- ゲージは現ステージ閾値から次ステージ閾値までの進捗率。両端に閾値（例 `10M [...] 30M`）を表示し、Stage 6 では MAX 表示。横幅節約のため中心軸は左寄せ（AXIS_COL=14）
- ペースは直近 1 時間の (時刻, 合計) サンプルのリングバッファから tokens/h を算出（3 分間隔なので最大 20 サンプル）
- セリフはステージ・ペース・時間帯（深夜など）に応じた候補プールからランダム選択し、60 秒ごとに入れ替え
- 必要サイズは最終形態（最大アート）を基準に固定する。幅 `maxContentWidth()+2`（現在30）、高さ `maxContentHeight()+1`（現在22）。一度この大きさに窓を合わせれば、成長してもメッセージは出ない。未満のときは「ウィンドウを広げて（よこ30 × たて22 いじょう）」と表示する

## エラー処理

- JSON.parse 失敗行はスキップ（書き込み途中の行を読む可能性があるため）
- オフセット > 現ファイルサイズ（削除・縮小）ならオフセットを 0 にリセットして読み直し
- ディレクトリ・ファイルの消失は無視して次の tick で再列挙
- スキャン中の例外は表示を巻き込まず、前回の値を表示し続ける

## テスト

collector と state の純粋部分を `node:test` でユニットテストする。

- 行パース: usage 抽出、壊れた行のスキップ、usage を含まない行の無視
- 日付キー化: UTC タイムスタンプ → JST 日付、日付またぎの境界
- 重複排除: 同一 message.id + requestId の二重カウント防止
- ステージ判定: 各閾値の境界値
- ペース計算: サンプル不足時（起動直後）の扱い

renderer は目視で確認する。

## 使い方

```
./clawd-pet.sh
```

専用ターミナルウィンドウで実行し、開いたままにする。終了は Ctrl+C。

## コスト計算

statusline の `total_cost_usd` と同じロジック。料金表と計算式は Claude Code 本体バイナリ（v2.1.173）から抽出した（USD per 1M tokens、cache write は 5m/1h で単価が異なる、web search は 1 件 $0.01）。未知モデルは opus 現行レート（$5/$25）にフォールバック、`<synthetic>` は $0。

## 進化エフェクト

ステージが上がった瞬間、本家 Claude Code の effort で ultracode を選んだときの波紋（UltraRippleText）を再現したエフェクトを約 4 秒再生する。

- ペット中心を原点に、距離 `√(dx² + (2·dy)²)` のコサイン波（波長 20 セル）が 0.03 セル/ms で拡大
- 波が覆ったセルは背景色を 8 段階の紫グラデーション `rgb(62,22,118)`→`rgb(140,80,240)`、文字色を白にする（定数はすべて本家バイナリと同値）
- 波紋中は 50ms 間隔で描画する。`--ripple` フラグで起動直後に試し見できる

## 設定と国際化（OSS化対応）

- 設定の優先順位: 環境変数 > `~/.config/clawd-pet/config.json` > システム既定（タイムゾーンは Intl、言語は LANG から推定）
- 初回起動時（設定ファイルがなく TTY のとき）にウィザードが言語（ja/en）とタイムゾーンを質問して保存する。EOF・中断は既定値で続行
- 言語: 表示テキスト（セリフ・ステージ名・ラベル）は `TEXTS` に ja/en を持つ
- タイムゾーン: 日付キーと0時計算は Intl ベースで任意の IANA タイムゾーンに対応（DST 込み）。`dayKeyJST` 等は Asia/Tokyo のラッパとして残す
- 進化閾値・スキャン間隔は設定可能。既定値は従来どおり
- `CLAUDE_CONFIG_DIR` で Claude Code の設定ディレクトリ移動に追従

## 料金表の自動更新

起動時に LiteLLM の公開 JSON から anthropic モデルの単価を取得して内蔵テーブルにマージする（per-token → per-1M 換算、1h キャッシュ単価がなければ慣例比率で補完）。失敗時は前回キャッシュ（`pricing-cache.json`）→ 内蔵テーブルの順にフォールバック。`CLAWD_PET_NO_FETCH=1` で無効化。

## 撫でる（マウスホイール）

ウィンドウ上でマウスホイールを回すと撫でられて喜ぶ。SGR マウストラッキング（`?1000h`+`?1006h`）を有効化し、stdin を raw mode にしてホイールイベント（SGR ボタン 64/65）を読む。`parseMouseEvents` で純粋にパースし、検出したら約 2.5 秒（`PET_DURATION_MS`）の撫でられ状態に入る。撫でられ中は目を細め（blink フレーム）、頭の右上にハートが舞い（`heartLines`、Zzz と同じスロット）、専用プールの喜びセリフを表示する。優先度は 撫でられ > 睡眠。raw mode のため Ctrl+C（`\x03`）と `q` を自前で終了に変換し、終了時にマウストラッキングを解除する。`--pet` で試し見できる。
